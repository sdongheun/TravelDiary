02Conventions

  • PK: 전 테이블 uuid DEFAULT gen_random_uuid() — 단, profiles.id는 auth.users.id 재사용, review_likes는 복합 PK.
  • Timestamps: 모든 테이블 created_at / updated_at timestamptz NOT NULL DEFAULT now() + set_updated_at() BEFORE UPDATE 트리거.
  • Scope: PRIVATE 본인만 · PUBLIC 공개읽기 · MASTER 공용 캐시. → RLS로 강제.
  • FK on delete: 소유 관계는 CASCADE, 참조 관계(place/diary)는 SET NULL로 기록 보존.
  • Legend: PK primary FK foreign UQ unique NN not null

06Design Notes

  • places = 공용 마스터 — 장소 메타데이터 중복 제거 + "동일 장소 후기 묶음"(혼행 선순환) 지원.
  • photos 1:N 분리 — 한 장소 다수 사진 + 사진별 EXIF 보존. records는 코스 단위로 슬림.
  • photos.user_id 비정규화 — 부모 JOIN 없이 auth.uid() = user_id 단일 조건 RLS.
  • records.diary_id 평면 FK — trips 중간 테이블 없이 일기 묶음(MVP 단순화).
  • answers jsonb — 동적 질문 유연성, 질문 텍스트까지 보존.
  • places writes = server-only — 클라이언트 키엔 write 정책 미부여, secret key로만 캐시 기록.
  • Migrations — 단일 init 파일, append-only. 변경 시 supabase migration new → supabase db push.
